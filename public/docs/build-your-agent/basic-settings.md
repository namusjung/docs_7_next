# Basic Settings

{% section id="basic-settings" title="Overview" %}

The Basic Settings section allows you to define the core identity and purpose of your chatbot agent. This is the first step in creating a new agent.

{% section id="configuration-fields" title="Configuration Fields" %}

| Field | Description | Example |
|-------|-------------|---------|
| **Agent Name** | The internal identifier for your agent, used across the platform for tracking and management. Should be unique and descriptive. | `Customer Support Bot` |
| **Display Name** | The name shown to end users during conversations. Should be friendly and brand-aligned. | `Support Buddy` |
| **Description** | A brief summary of what the agent does. Helps your team understand the agent's purpose at a glance. | `Handles customer support inquiries for the billing department` |

{% callout type="tip" title="Best Practices for Basic Settings" collapsible=true %}

| Setting         | Best Practice                                                                 |
|-----------------|-------------------------------------------------------------------------------|
| **Agent Name**  | Choose a clear and descriptive name for internal tracking (e.g., `Customer Support Bot`). |
| **Display Name**| Set a friendly, brand-aligned name for user interaction (e.g., `Support Buddy`).       |
| **Description** | Summarize the agent’s purpose concisely, such as `Handles customer support inquiries`. |

{% /callout %}
